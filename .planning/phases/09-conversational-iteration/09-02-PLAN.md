# Plan: 09-02 Version History & Undo

## Objective

Let users browse design version history within a conversation and revert to previous versions.

## Context

Design versions are tracked in the database (version field, parent_design_id) but the UI treats each design atomically. This plan adds version navigation so users can compare and revert.

## Tasks

- [ ] **Task 1: Version history API**
      Add `GET /api/conversations/{id}/design-history` returning all designs for a conversation ordered by version. Include parameters, version number, created_at, and a flag for which is current. Add to conversations router.

- [ ] **Task 2: Version history UI**
      Create `VersionHistory.tsx` component — compact timeline showing version numbers with timestamps. Clicking a version loads its parameters into preview. Show as a collapsible panel in the preview header area. Highlight current version.

- [ ] **Task 3: Revert to version**
      Add `POST /api/conversations/{id}/revert/{design_id}` that creates a new design version copying the target's parameters (don't delete history — append). Wire revert button in VersionHistory. On revert, trigger re-generation with the restored parameters.

## Notes

- Revert creates a NEW version (v4 = copy of v2's params), preserving full history
- No destructive operations — all versions kept
- Version history only shown when version > 1
