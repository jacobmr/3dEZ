**Modify `ConversationService._build_api_messages()` in `backend/src/app/services/conversation.py`:**

- Accept `db` session parameter (needed to load photo files)
- When a user message has `photo_ids`, build multi-part content:
  ```python
  if msg.role == "user":
      if msg.photo_ids:
          content_blocks = [{"type": "text", "text": msg.content}]
          for photo_id in msg.photo_ids:
              # Load photo from disk, base64 encode
              photo = await db.get(Photo, photo_id)
              if photo:
                  image_data = Path(photo.file_path).read_bytes()
                  b64 = base64.b64encode(image_data).decode()
                  content_blocks.append({
                      "type": "image",
                      "source": {
                          "type": "base64",
                          "media_type": photo.content_type,
                          "data": b64,
                      }
                  })
          api_msgs.append({"role": "user", "content": content_blocks})
      else:
          api_msgs.append({"role": "user", "content": msg.content})
  ```
- Make \_build_api_messages an async method (needs file I/O)
- Update all callers of \_build_api_messages to await it and pass db session

## Task 2: Vision system prompt and analyze_photo tool

type: auto

**Add `analyze_photo` tool to `backend/src/app/models/tools.py`:**

```python
{
    "name": "analyze_photo",
    "description": "Analyze a user's uploaded photo to extract physical context for 3D design. Call this when the user uploads a photo of where their design will be used.",
    "input_schema": {
        "type": "object",
        "properties": {
            "environment": {
                "type": "string",
                "description": "Type of environment (e.g., 'desk setup', 'workshop wall', 'kitchen cabinet', 'garage shelf')"
            },
            "surface_material": {
                "type": "string",
                "description": "Material of mounting/placement surface (e.g., 'drywall', 'wood', 'metal', 'tile')"
            },
            "reference_objects": {
                "type": "array",
                "items": {
                    "type": "object",
                    "properties": {
                        "name": {"type": "string", "description": "Object name (e.g., 'US wall outlet', 'USB-A port', 'M4 screw')"},
                        "known_dimension_mm": {"type": "number", "description": "Known real-world dimension in mm"},
                        "dimension_type": {"type": "string", "enum": ["width", "height", "diameter"]},
                    },
                    "required": ["name", "known_dimension_mm", "dimension_type"]
                },
                "description": "Objects with known dimensions visible in the photo"
            },
            "nearby_objects": {
                "type": "array",
                "items": {"type": "string"},
                "description": "Other objects visible that provide context (e.g., 'monitor', 'keyboard', 'power strip')"
            },
            "suggested_constraints": {
                "type": "string",
                "description": "Spatial constraints inferred from the photo (e.g., 'must fit between outlet and shelf edge, roughly 80mm gap')"
            }
        },
        "required": ["environment", "reference_objects"]
    }
}
```

**Update system prompt in `backend/src/app/prompts/design_wizard.py`:**

Add a PHOTO ANALYSIS section to the system prompt:

```
## PHOTO ANALYSIS

When the user uploads a photo:
1. ALWAYS call the analyze_photo tool to extract context
2. Look for reference objects with known dimensions:
   - US standard wall outlet: 70mm wide × 115mm tall
   - US light switch plate: 70mm wide × 115mm tall
   - USB-A port: 12mm wide × 4.5mm tall
   - Standard credit card: 85.6mm × 53.98mm
   - US quarter coin: 24.26mm diameter
   - AA battery: 14.5mm diameter × 50.5mm long
   - M3 screw head: 5.5mm diameter
   - M4 screw head: 7mm diameter
3. Use reference objects to estimate dimensions of the space/area
4. Suggest appropriate dimensions based on the physical context
5. Always confirm inferred dimensions with the user before finalizing
```
