<task type="auto">
  <name>Task 2: Create docker-compose.prod.yml with Caddy reverse proxy and env template</name>
  <files>docker-compose.prod.yml, Caddyfile, .env.production.example</files>
  <action>
**docker-compose.prod.yml** — Production compose file with 3 services:
- `backend`: Build from backend/Dockerfile, expose NO ports externally (internal network only), env_file .env.production, healthcheck on /api/health, restart: unless-stopped
- `frontend`: Build from frontend/Dockerfile, expose NO ports externally (internal network only), depends_on backend (healthy), restart: unless-stopped
- `caddy`: Use caddy:2-alpine image, ports 80:80 and 443:443, mount ./Caddyfile:/etc/caddy/Caddyfile, mount caddy_data and caddy_config volumes, depends_on frontend and backend, restart: unless-stopped
- Named volumes: caddy_data, caddy_config (for HTTPS cert persistence)
- Default network connecting all services

**Caddyfile** — Caddy reverse proxy config:
```
ez3d.salundo.com {
    handle /api/* {
        reverse_proxy backend:8000
    }
    handle {
        reverse_proxy frontend:3000
    }
}
```
Caddy auto-provisions Let's Encrypt HTTPS. No manual cert management needed.

**env.production.example** — Template with required env vars:
```
ANTHROPIC_API_KEY=sk-ant-...
DATABASE_URL=sqlite+aiosqlite:///./data/3dez.db
```
Note: SQLite path should be a Docker volume mount for data persistence. Add a `db_data` volume in compose mapped to /app/data in backend.

Also add a `make deploy` target to Makefile:
```makefile
deploy:
	docker compose -f docker-compose.prod.yml up -d --build

deploy-down:
	docker compose -f docker-compose.prod.yml down

deploy-logs:
	docker compose -f docker-compose.prod.yml logs -f
```
