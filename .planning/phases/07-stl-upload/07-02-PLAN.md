# Plan: 07-02 STL Modification Flow

## Objective

Enable conversation-driven modifications to uploaded STL files using mesh boolean operations (manifold3d).

## Context

After 07-01, users can upload STLs and Claude can analyze them. This plan adds the ability to actually modify uploaded meshes: add features (union), cut holes (difference), and trim shapes (intersection) using parametric primitives generated by the existing modeler.

## Tasks

- [x] **Task 1: manifold3d dependency & mesh_ops module**
      Add manifold3d to Dockerfile.dev and Dockerfile (prod). Create `backend/src/app/modeler/mesh_ops.py` with `boolean_stl(base_stl: bytes, tool_stl: bytes, operation: str) -> bytes` using trimesh's manifold3d integration. Add validation on the result.

- [x] **Task 2: modify_stl tool & conversation integration**
      Add `modify_stl` tool to Claude tools. Input: stl_file_id, modification_type (add_feature/cut_hole/trim), primitive_params (shape, dimensions, position). Handle in conversation service — generate primitive STL via ModelEngine, run boolean op, save result as new StlFile, yield modified STL event.

- [x] **Task 3: Frontend modification preview**
      When a modified STL is returned, update the Three.js preview to show the result. Add before/after toggle. Show modification description from Claude's response.

## Notes

- Only watertight-to-watertight operations (both inputs must be manifold)
- Parametric primitives (box, cylinder, etc.) are always watertight by construction
- Keep modification scope limited: union, difference, intersection with simple primitives
- Complex mesh editing (fillets, face extrusion) is out of scope — requires BRep
