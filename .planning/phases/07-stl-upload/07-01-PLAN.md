# Plan: 07-01 STL Upload & Analysis

## Objective

Allow users to upload existing STL files, validate them, and surface metadata to Claude for conversation-driven modifications.

## Context

The app has photo upload as a template (photos.py). STL files need validation (watertight, manifold), size limits, and metadata extraction. Claude needs an `analyze_imported_stl` tool to reason about uploaded geometry. The Three.js preview already handles STL display.

## Tasks

- [x] **Task 1: StlFile model & migration**
      Add `StlFile` model to `db/models.py` with fields: id, session_id, conversation_id, filename, file_path, content_type, file_size, vertex_count, face_count, is_watertight, bounding_box (JSON), created_at. Add SQLite migration in engine.py. Storage at `data/stl/{session_id}/`.

- [x] **Task 2: STL upload endpoint**
      Create `backend/src/app/api/stl_files.py` with `POST /api/conversations/{id}/stl-files`. Accept multipart upload, 25MB limit, validate with trimesh (watertight check, face count < 500k), extract metadata, store file, return metadata response. Register router in main.py.

- [x] **Task 3: analyze_imported_stl Claude tool**
      Add `analyze_imported_stl` tool definition to `models/tools.py`. Input schema: stl_file_id, dimensions (width/height/depth mm), face_count, is_watertight, suggested_modifications. Handle tool use in conversation service — yield `stl_analysis` event. Update system prompt to mention STL upload capability.

- [x] **Task 4: Frontend STL upload component**
      Add STL upload button to chat input area (alongside photo upload). Accept .stl files, show file size, upload to endpoint, display bounding box in preview panel. Send stl_file_id with message to conversation.
