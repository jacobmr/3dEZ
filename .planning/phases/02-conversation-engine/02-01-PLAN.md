Create backend/src/app/db/ package:

engine.py:

- Async SQLAlchemy engine and session factory using config DATABASE_URL
- get_db() async dependency for FastAPI (yields AsyncSession)
- create_tables() async function to create all tables on startup

models.py using SQLAlchemy ORM (DeclarativeBase):

- Session table: id (UUID primary key), created_at, last_active_at. This represents a user session (anonymous for V1 — no auth, just a session token stored client-side).
- Conversation table: id (UUID), session_id (FK to Session), title (nullable, auto-generated from first message), status (active/completed), created_at, updated_at.
- Message table: id (UUID), conversation_id (FK), role (user/assistant/system), content (text), tool_use (JSON nullable — for storing tool call data), created_at.
- Design table: id (UUID), conversation_id (FK), parameters (JSON — the extracted design parameters), category (string), version (int, default 1 — increments on revision), created_at.

All UUIDs generated with uuid4. Use server_default for timestamps.

Update main.py: Add lifespan event handler that calls create_tables() on startup. Import and register the db dependency.

Multi-tenancy: All queries filter by session_id. Sessions are created on first request and identified by a token in X-Session-ID header (generated client-side as UUID, validated server-side).

Do NOT add alembic/migrations — create_tables() with checkfirst=True is fine for V1.
</action>
