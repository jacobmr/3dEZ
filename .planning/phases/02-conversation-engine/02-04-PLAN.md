Create backend/src/app/api/conversations.py with APIRouter(prefix="/api/conversations"):

POST / — Start new conversation

- Depends on session_id, db
- Body: { message: str } (initial user message)
- Creates conversation via ConversationService.start_conversation()
- Returns: { conversation_id, title: null, status: "active" }

GET / — List conversations for session

- Depends on session_id, db
- Query params: limit (int=20), offset (int=0)
- Returns: list of ConversationSummary objects

GET /{conversation_id} — Get conversation with messages

- Depends on session_id, db
- Returns: { conversation, messages: [...], latest_design: {...} | null }
- 404 if conversation doesn't belong to this session

DELETE /{conversation_id} — Delete conversation

- Depends on session_id, db
- Cascade delete messages and designs
- 204 No Content

Create backend/src/app/api/designs.py with APIRouter(prefix="/api/designs"):

GET / — List saved designs for session

- Depends on session_id, db
- Returns designs with their conversation titles
- Ordered by created_at desc

GET /{design_id} — Get design details

- Returns full parameter set + conversation context

Register both routers in main.py.

<task type="auto">
  <name>Task 2: Create SSE streaming endpoint</name>
  <files>backend/src/app/api/conversations.py</files>
  <action>
Add SSE streaming endpoints to the conversations router:

POST /{conversation_id}/message — Send message and stream response

- Depends on session_id, db
- Body: { message: str }
- Returns: StreamingResponse with content-type text/event-stream
- Streams events from ConversationService.send_message():
  - event: text, data: {"content": "..."} (text chunks)
  - event: clarification, data: {"question": "...", "options": [...]}
  - event: parameters_extracted, data: {"parameters": {...}, "design_id": "..."}
  - event: error, data: {"message": "..."}
  - event: done, data: {} (stream complete)
- Format as SSE: "event: {type}\ndata: {json}\n\n"

POST /{conversation_id}/revise — Revise existing design

- Same streaming format as /message
- Body: { message: str } (revision request like "make it 3mm deeper")
- Uses ConversationService.revise_design()
- Streams same event types

Use FastAPI StreamingResponse with media_type="text/event-stream". Set headers: Cache-Control: no-cache, Connection: keep-alive.

Handle client disconnect gracefully — if the client disconnects mid-stream, catch the error and don't leave partial state in DB.

Do NOT use WebSocket — SSE is simpler and sufficient for server→client streaming. Client→server is regular POST.
</action>
