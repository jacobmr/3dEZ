<task type="auto">
  <name>Task 1: Create shared API type definitions</name>
  <files>shared/api-types.ts, backend/src/app/models/api.py, Makefile</files>
  <action>
Create shared type definitions that keep frontend and backend in sync:

- `shared/api-types.ts` — TypeScript interfaces for API contracts. Start with: `HealthResponse { status: string; service: string }`, `ConversationMessage { role: "user" | "assistant"; content: string; timestamp: string }`, `DesignParameters { category: string; dimensions: Record<string, number>; features: Record<string, unknown> }`. These are the foundational types that Phase 2+ will extend.
- `backend/src/app/models/api.py` — Matching Pydantic v2 models for the same contracts. Use `BaseModel` with field descriptions. These must mirror the TypeScript types exactly.
- `Makefile` at project root with targets:
  - `dev`: `docker compose up -d` (start everything)
  - `dev-down`: `docker compose down`
  - `dev-logs`: `docker compose logs -f`
  - `backend-shell`: `docker compose exec backend bash`
  - `frontend-shell`: `docker compose exec frontend sh`
  - `lint`: Run ruff (backend) and eslint (frontend) in parallel
  - `test`: Run pytest (backend) and npm test (frontend) in parallel
  - `build`: Build both services
  - `clean`: Remove containers, volumes, caches

Keep types minimal — only define what's needed now. Phase 2 will add conversation-specific types.
  </action>
  <verify>cd backend && python -c "from app.models.api import HealthResponse; print(HealthResponse(status='ok', service='test').model_dump())" && cd ../frontend && npx tsc --noEmit</verify>
  <done>Pydantic models instantiate correctly, TypeScript types compile, Makefile targets work</done>

<task type="auto">
  <name>Task 2: Set up CI pipeline and project README</name>
  <files>.github/workflows/ci.yml, .gitignore, README.md</files>
  <action>
Create CI and project scaffolding:

- `.github/workflows/ci.yml` — GitHub Actions workflow triggered on push/PR to main:
  - Job 1 (backend): Python 3.11, install deps from pyproject.toml, run `ruff check backend/`, run `mypy backend/src/` (if configured), run `pytest backend/` (allow empty for now).
  - Job 2 (frontend): Node 20, npm ci in frontend/, run `npm run lint`, run `npm run build`.
  - Both jobs run in parallel.
  - Do NOT try to install OCP in CI — it requires system deps. Skip OCP-dependent tests via marker.
- `.gitignore` — Comprehensive gitignore for Python (__pycache__, .venv, *.pyc, .mypy_cache, .ruff_cache, .pytest_cache), Node (node_modules, .next, .env.local), Docker (.env), IDE (.vscode, .idea), OS (.DS_Store).
- `README.md` — Brief project description, prerequisites (Docker, Node 20, Python 3.11), quickstart (`make dev`), project structure overview, development workflow. Keep it practical, not marketing.

CI should be lightweight and fast. No Docker builds in CI — just lint and build checks.
  </action>
