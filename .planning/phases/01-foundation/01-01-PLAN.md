Create `backend/` directory with Python project structure:

- `pyproject.toml` with project metadata, dependencies (fastapi, uvicorn, python-multipart, pydantic>=2), dev dependencies (pytest, ruff, mypy). Use `[project]` format (PEP 621). Python >=3.11. Set `[tool.ruff]` for linting config (line-length=100, target-version="py311").
- `src/app/main.py` — FastAPI app instance with CORS middleware (allow all origins for dev), include health router.
- `src/app/api/health.py` — GET `/api/health` endpoint returning `{"status": "ok", "service": "3dez-backend"}`.
- `src/app/__init__.py` and `src/app/api/__init__.py` — empty init files.

Do NOT add OCP dependencies to pyproject.toml yet — they require system libraries and will be handled via Docker in Task 2. Keep pyproject.toml dependencies to pure Python packages only.
  </action>
  <verify>cd backend && pip install -e ".[dev]" && python -c "from app.main import app; print('OK')"</verify>
  <done>FastAPI app imports cleanly, pyproject.toml is valid, project structure follows src layout</done>

<task type="auto">
  <name>Task 2: Create Docker dev environment with OCP</name>
  <files>backend/Dockerfile, backend/Dockerfile.dev, docker-compose.yml, backend/.dockerignore</files>
  <action>
Create Docker setup for backend development:

- `backend/Dockerfile.dev` — Development Dockerfile based on `python:3.11-slim`. Install system deps for OCP: `libgl1-mesa-glx libglu1-mesa libx11-6 libxext6 libxrender1`. Install `cadquery` and `build123d` via pip (these bundle OCP). Install project in editable mode. CMD runs uvicorn with `--reload` on port 8000.
- `backend/Dockerfile` — Production Dockerfile (multi-stage). Same OCP deps but no dev dependencies, no --reload. Copy built wheel. CMD runs uvicorn on port 8000.
- `backend/.dockerignore` — Exclude `.venv`, `__pycache__`, `.pytest_cache`, `.mypy_cache`, `.ruff_cache`.
- `docker-compose.yml` at project root — Service `backend` using Dockerfile.dev, port mapping 8000:8000, volume mount `./backend/src:/app/src` for hot reload, healthcheck on `/api/health`.

Use build123d (not raw OCP) — it provides a clean Pythonic API on top of OpenCascade. cadquery is the alternative but build123d is newer and more actively maintained. Install both for now: `pip install build123d cadquery`.

Verify OCP is accessible inside container: `python -c "import OCP; print('OCP OK')"`.
  </action>
  <verify>docker compose build backend && docker compose up -d backend && sleep 3 && curl -s http://localhost:8000/api/health | grep '"status":"ok"' && docker compose exec backend python -c "import OCP; print('OCP OK')"</verify>
  <done>Backend container builds, health endpoint responds on localhost:8000, OCP imports successfully inside container</done>
</task>
