# Plan: 10-02 Mobile UX Polish

## Objective

Optimize the mobile experience for responsive layout, touch interactions, and performance.

## Context

The app works on mobile but hasn't been specifically optimized. The 40/60 desktop split needs mobile adaptation, touch targets need sizing, and performance on lower-end devices needs attention.

## Tasks

- [ ] **Task 1: Responsive layout**
      On mobile (<768px), stack conversation above preview instead of side-by-side. Add swipe or tab navigation between chat and preview views. Preview should be accessible without scrolling past all messages. Add a floating "View Preview" button when in chat mode.

- [ ] **Task 2: Touch-friendly controls**
      Increase touch targets to 44px minimum for all interactive elements. Improve orbit controls sensitivity for touch. Make version history and parameter nudge buttons thumb-friendly. Ensure suggested modification chips don't overflow on narrow screens.

- [ ] **Task 3: Performance optimization**
      Lazy-load Three.js and preview components (dynamic import). Reduce initial bundle size by code-splitting chat vs preview. Add `loading="lazy"` to non-critical images. Minimize re-renders in message list with React.memo. Test and optimize for devices with limited WebGL support.

## Notes

- Mobile-first approach for new components going forward
- WebGL performance varies significantly across mobile devices
- Touch orbit controls already exist via @react-three/drei, just need tuning
