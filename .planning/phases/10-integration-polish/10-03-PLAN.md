# Plan: 10-03 Error Handling & Edge Cases

## Objective

Add comprehensive error handling, retry logic, and graceful failure modes throughout the application.

## Context

The app handles happy paths well but needs robust error handling for production use: API failures, network issues, invalid inputs, and edge cases in the modeler.

## Tasks

- [ ] **Task 1: API error handling**
      Add consistent error response format across all endpoints (error code, message, details). Add global error boundary in React app. Show user-friendly error messages in chat when Claude API fails, when generation fails, or when network is lost. Add retry button for failed operations.

- [ ] **Task 2: Input validation**
      Add comprehensive input validation on design parameters (min/max ranges for dimensions, valid category checking). Validate conversation message length limits. Add rate limiting on API endpoints (10 messages/minute, 5 generations/hour). Show validation errors inline in chat.

- [ ] **Task 3: Graceful degradation**
      Handle WebGL not supported (show message with download-only option). Handle Claude API rate limits with exponential backoff. Add connection status indicator. Handle concurrent modification conflicts (two tabs editing same design). Add auto-save for long conversations.

## Notes

- Error messages should be helpful, not technical
- Rate limits prevent abuse while allowing normal usage
- Graceful degradation > hard failures
