# Plan: 10-01 End-to-End Flow & STL Download

## Objective

Ensure the complete flow from conversation to STL download works seamlessly, with proper state management across all features.

## Context

All individual features are built (conversation, modeler, preview, auth, cost estimation, iteration). This plan connects them into a cohesive end-to-end experience and ensures the STL download path is robust.

## Tasks

- [ ] **Task 1: Download flow polish**
      Ensure the STL download button is prominent and functional across all design paths: new design, revised design, uploaded+modified STL. Add filename based on design category and version (e.g., "bracket_v2.stl"). Add download count tracking on Design model. Verify download works on mobile browsers.

- [ ] **Task 2: New conversation flow**
      Polish the "new design" flow: welcome message with category suggestions, smooth transition from empty state to first design. Add a "New Design" button in the sidebar/header that creates a fresh conversation. Ensure previous conversation state fully clears.

- [ ] **Task 3: Design sharing**
      Add `GET /api/designs/{id}/share` that returns a shareable link with a unique token. Unauthenticated users can view (but not modify) shared designs. Create a minimal `SharedDesignView` page that loads and previews the STL. Add "Share" button next to download.

## Notes

- Focus on user-facing flow quality, not infrastructure
- Download filename should be descriptive and unique
- Share links are read-only, no auth required to view
