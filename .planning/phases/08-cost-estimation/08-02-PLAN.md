# Plan: 08-02 Pricing UX & Approval Flow

## Objective

Display cost estimates to users before STL generation and add an approval gate.

## Context

After 08-01, the backend can calculate costs. This plan adds the frontend flow: show estimated price before generating, let user approve, track usage per user.

## Tasks

- [x] **Task 1: Cost estimate display component**
      Create `CostEstimate.tsx` component showing price breakdown (conversation cost, generation cost, total with markup). Display in chat when design parameters are extracted, before generation. Style as a card with approve/decline buttons.

- [x] **Task 2: Approval flow integration**
      Wire approve button to `POST /api/conversations/{id}/approve-cost`. On approval, trigger STL generation. On decline, let user continue modifying. Add `cost_approved` field to Design model. Update generate endpoint to check approval.

- [x] **Task 3: Usage tracking dashboard**
      Add `GET /api/users/me/usage` endpoint returning total designs generated, total cost, usage this month. Display basic usage stats in user profile/settings area.

## Notes

- V1: display-only pricing (no actual payment integration)
- Cost approval is a soft gate — informs user but doesn't block
- Usage tracking enables future billing integration
