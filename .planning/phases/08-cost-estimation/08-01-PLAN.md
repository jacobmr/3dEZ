# Plan: 08-01 Cost Estimation Engine

## Objective

Build a cost estimation system that calculates COGS (Claude API tokens + compute time) and applies a 2x markup for pricing.

## Context

The app generates 3D designs via Claude conversation + OCP modeling. We need to track costs per design generation: Claude API token usage, compute time for STL generation, and display estimated price to users before generating.

## Tasks

- [x] **Task 1: Token usage tracking**
      Add `token_usage` JSON column to Message model (prompt_tokens, completion_tokens, total_tokens). Capture usage from Claude API response in conversation service. Add migration in engine.py.

- [x] **Task 2: Cost calculation service**
      Create `backend/src/app/services/cost_estimation.py` with `estimate_cost(conversation_id)` that calculates: total tokens used, estimated generation compute cost (fixed per category), COGS total, and 2x markup price. Return `CostEstimate` dataclass with breakdown.

- [x] **Task 3: Cost estimation API endpoint**
      Add `GET /api/conversations/{id}/cost-estimate` to return cost breakdown before generation. Add `POST /api/conversations/{id}/approve-cost` for user to approve before STL generation proceeds.

## Notes

- Claude API pricing: ~$3/M input tokens, ~$15/M output tokens (Sonnet 3.5)
- OCP compute cost: flat rate per category (bracket ~$0.05, enclosure ~$0.08, organizer ~$0.06)
- 2x markup on total COGS for user-facing price
- Track but don't block — V1 shows estimate, doesn't require payment
