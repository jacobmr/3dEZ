# Plan: 06-03 Design Library UI

## Objective

User's saved designs as a proper library with duplicate/variant creation and design management.

## Context

Currently the sidebar shows conversations as "designs" (1:1 mapping). Need to add:

- Design name field (independent of conversation title)
- Duplicate design endpoint (creates new conversation with pre-filled parameters)
- Design library view with category filtering
- Variant lineage tracking (parent_design_id)

## Tasks

- [ ] **Task 1: Design model enhancements**
      Add `name` (nullable, defaults to conversation title), `parent_design_id` (nullable FK to self for variants) to Design model. Add `POST /api/designs/{id}/duplicate` endpoint that creates a new conversation + design with copied parameters.

- [ ] **Task 2: Design API enhancements**
      Add query params to `GET /api/designs/`: `category` filter, `search` text (name/title), `sort` (created_at desc/asc). Add `PATCH /api/designs/{id}` for renaming. Add `GET /api/designs/{id}/variants` to list child designs.

- [ ] **Task 3: Design library frontend**
      Replace conversation-based sidebar list with design-based library. Add category filter tabs, search input, duplicate button on design cards, rename inline editing. Show variant count badge on designs that have children.

## Notes

- Keep conversation as the unit of interaction — duplicating a design creates a new conversation
- Design name falls back to conversation title if null
- Variant lineage is informational only (no tree view in V1)
