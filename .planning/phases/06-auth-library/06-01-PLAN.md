# Phase 6, Plan 1: Auth System

<objective>
Add email+password authentication with JWT tokens. Create User model, auth endpoints, JWT access+refresh flow, frontend auth UI. Maintain backward compatibility with anonymous sessions.
</objective>

<context>
- Current identity: `X-Session-ID` header with localStorage UUID
- Single identity extraction: `get_session_id` in `backend/src/app/api/deps.py`
- Single header injection: `headers()` in `frontend/src/lib/api.ts`
- All data linked through Session table via session_id FK
- Approach: email+password with self-hosted JWT (no external auth providers)
</context>

<tasks>

## Task 1: Add auth dependencies and User model

type: auto

Add `pyjwt>=2.8`, `passlib[bcrypt]>=1.7`, `bcrypt>=4.0` to `backend/pyproject.toml`.

Add JWT config to `backend/src/app/core/config.py`:

- JWT_SECRET_KEY, JWT_ALGORITHM, JWT_ACCESS_TOKEN_EXPIRE_MINUTES (15), JWT_REFRESH_TOKEN_EXPIRE_DAYS (7)

Add User model to `backend/src/app/db/models.py`:

- id (UUID PK), email (unique, indexed), hashed_password, created_at
- Add user_id nullable FK to Session model with relationship

Files: backend/pyproject.toml, backend/src/app/core/config.py, backend/src/app/db/models.py

## Task 2: Auth service — password hashing and JWT

type: auto

Create `backend/src/app/services/auth.py`:

- hash_password / verify_password (passlib CryptContext bcrypt)
- create_access_token / create_refresh_token (PyJWT with exp claims)
- decode_token with type validation
- register_user (create user + optional session claiming)
- authenticate_user (verify credentials)
- claim_session (link anonymous session to user)

Files: backend/src/app/services/auth.py

## Task 3: Auth API endpoints

type: auto

Create `backend/src/app/api/auth.py`:

- POST /api/auth/register — create user, return tokens, set refresh cookie
- POST /api/auth/login — verify credentials, return tokens, set refresh cookie
- POST /api/auth/refresh — rotate tokens via cookie
- GET /api/auth/me — return current user profile
- Register in main.py

Files: backend/src/app/api/auth.py, backend/src/app/main.py

## Task 4: Dual-mode auth dependency

type: auto

- Keep get_session_id as thin wrapper for backward compat

Update ownership checks in conversations.py, designs.py, photos.py to support multi-session via RequestContext.

Files: backend/src/app/api/deps.py, backend/src/app/api/conversations.py, backend/src/app/api/designs.py, backend/src/app/api/photos.py
