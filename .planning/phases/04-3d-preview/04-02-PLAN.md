# Phase 4, Plan 2: Dimension Overlays

<objective>
Display key measurements on 3D models using dimension lines with labels. Build reusable DimensionLine component from drei primitives (Line + Text + Billboard) and a DimensionOverlay that maps design parameters to positioned annotations.
</objective>

<execution_context>

- drei Line, Text, Billboard components for 3D annotations
- No npm package for CAD-style dimension lines — custom component needed
- Design params contain actual dimensions (width, height, depth, wall_thickness, etc.)
- Bounding box from STL geometry provides anchor positions
  </execution_context>

<context>
Key files:
- frontend/src/components/preview/StlMesh.tsx — STL loading, geometry centering (from 04-01)
- frontend/src/components/preview/StlViewer.tsx — scene composition (from 04-01)
- frontend/src/components/preview/PreviewPanel.tsx — container (from 04-01)
- shared/design-params.ts — MountingBracketParams, EnclosureParams, OrganizerParams

Dimension line anatomy:

- Two end points (arrows/ticks)
- Connecting line between them
- Label at midpoint (always camera-facing via Billboard)
- Offset from model surface to avoid overlap
  </context>

<tasks>
## Task 1: Create DimensionLine component

A single dimension annotation that draws a measurement between two 3D points.

**Files to create:**

1. `frontend/src/components/preview/DimensionLine.tsx`
   - Props: `start: [number, number, number]`, `end: [number, number, number]`, `label: string`, `color?: string`
   - Draw using drei `<Line>` for the main line and tick marks at endpoints
   - Tick marks: short perpendicular lines at start and end points
   - Label: drei `<Text>` inside `<Billboard>` at midpoint, always facing camera
   - Text styling: small font (fontSize ~2-3), high contrast color
   - Extension lines: thin lines from dimension line to the model surface (offset from model)

**Verification:**

- `cd frontend && npx tsc --noEmit` passes
- Component renders without errors when given two 3D points

## Task 2: Create DimensionOverlay and integrate into StlViewer

Map design parameters to dimension annotations positioned on the model's bounding box.

**Files to create:**

1. `frontend/src/components/preview/DimensionOverlay.tsx`
   - Props: `category: string`, `params: Record<string, unknown>`, `boundingBox: { min: [number,number,number], max: [number,number,number] }`
   - Map known params to DimensionLine positions:
     - **width**: line along X axis (bottom edge, offset below)
     - **height**: line along Y axis (side edge, offset right)
     - **depth**: line along Z axis (bottom edge, offset front)
     - **wall_thickness**: shorter line showing wall detail (if applicable)
   - Format labels: value + "mm" unit suffix
   - Only show dimensions that exist in the params dict
   - Offset dimension lines ~5-10% beyond bounding box so they don't overlap mesh

**Files to modify:**

2. `frontend/src/components/preview/StlMesh.tsx`
   - After computing bounding box, expose min/max via a callback prop or ref
   - Add `onBoundsComputed?: (bounds: { min: [number,number,number], max: [number,number,number] }) => void`

3. `frontend/src/components/preview/StlViewer.tsx`
   - Accept optional `category` and `params` props
   - Track bounding box state from StlMesh callback
   - Render `<DimensionOverlay>` when bounds and params are available

4. `frontend/src/components/preview/PreviewPanel.tsx`
   - Pass `category` and `params` through to StlViewer when available

**Verification:**

- `cd frontend && npx tsc --noEmit` passes
- `cd frontend && npm run build` succeeds
- DimensionOverlay produces correct line positions from bounding box
