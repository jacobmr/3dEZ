# Phase 4, Plan 3: Preview Integration

<objective>
Connect the parametric modeler output to the 3D renderer. When a design is extracted from conversation, automatically generate STL and display it in the preview panel. Handle loading states, errors, and regeneration on parameter changes.
</objective>

<execution_context>
- POST /api/generate accepts {category, parameters}, returns binary STL bytes
- useConversation hook has currentDesign: {params, id} which updates on parameters_extracted SSE event
- PreviewPanel accepts stlBytes, isLoading, error props (from 04-01)
- API proxy: frontend :3000/api/* → backend :8000/api/* (configured in next.config.ts)
- 503 response when build123d not available (non-Docker environment)
</execution_context>

<context>
Key files:
- frontend/src/lib/api.ts — API helpers with session headers
- frontend/src/hooks/useConversation.ts — currentDesign state
- frontend/src/components/HomeClient.tsx — wires everything together
- frontend/src/components/preview/PreviewPanel.tsx — preview container (from 04-01)
- backend/src/app/api/generate.py — POST /api/generate endpoint
</context>

<tasks>
## Task 1: Add STL generation API client and usePreview hook

Create the API function and a hook to manage STL generation state.

**Files to modify:**

1. `frontend/src/lib/api.ts`
   - Add `generateStl(category: string, parameters: Record<string, unknown>): Promise<ArrayBuffer>`
   - POST to `/api/generate` with JSON body `{category, parameters}`
   - Return `response.arrayBuffer()`
   - Handle errors (400 bad category, 422 invalid params, 503 modeler unavailable)

**Files to create:**

2. `frontend/src/hooks/usePreview.ts`
   - Accepts: `design: { params: DesignParams; id: string } | null`
   - State: `stlBytes: ArrayBuffer | null`, `isLoading: boolean`, `error: string | null`
   - useEffect: when design changes (compare by id or params hash), trigger STL generation
   - Call `generateStl(design.params.category, design.params)` — extract category from params
   - Set loading → call API → set bytes or error
   - Clear stlBytes when design becomes null
   - Return `{ stlBytes, isLoading, error, regenerate }`
   - `regenerate()` function to manually retry

**Verification:**
- `cd frontend && npx tsc --noEmit` passes

## Task 2: Wire preview into HomeClient and add UX polish

Connect usePreview to HomeClient and PreviewPanel for the full flow.

**Files to modify:**

1. `frontend/src/components/HomeClient.tsx`
   - Import and use `usePreview(currentDesign)`
   - Pass `stlBytes`, `isLoading`, `error` to PreviewPanel
   - Pass `category` and `params` from currentDesign for dimension overlays
   - Remove the old JSON params display (or keep as collapsible detail below preview)

2. `frontend/src/components/preview/PreviewPanel.tsx`
   - Refine loading state: show "Generating 3D model..." with spinner
   - Refine error state: show error message + "Retry" button
   - For 503 errors: show "3D preview requires Docker environment" message
   - Add subtle transition/fade when new model loads
