  <name>Task 1: Create mesh validation module with trimesh</name>
  <files>backend/src/app/modeler/validation.py, backend/Dockerfile.dev</files>
  <action>
1. Add `trimesh` to Dockerfile.dev pip install line (alongside build123d, cadquery).

2. Create `modeler/validation.py` with:

`validate_mesh(stl_bytes: bytes) -> ValidationResult`:

- Load STL bytes into trimesh mesh: `trimesh.load(io.BytesIO(stl_bytes), file_type='stl')`
- Check `mesh.is_watertight` — mesh has no holes
- Check `mesh.is_winding_consistent` — all face normals point consistently outward
- Check `mesh.volume > 0` — positive volume (not inside-out)
- Return a `ValidationResult` dataclass with: is_valid (bool), is_watertight (bool), is_manifold (bool), volume (float), extents (tuple[float,float,float]), warnings (list[str])

`validate_dimensions(stl_bytes: bytes, expected: dict[str, float], tolerance: float = 0.2) -> DimensionResult`:

- Load mesh, get `mesh.extents` (size_x, size_y, size_z)
- Compare each expected dimension against mesh extents (sorted, since axis orientation may vary)
- Flag any dimension off by more than `tolerance` mm
- Return `DimensionResult` dataclass with: is_accurate (bool), expected (dict), actual (dict), deviations (dict)

3. Integrate into `ModelEngine.generate()`:

- After STL export, run `validate_mesh()` on the bytes
- If not valid, raise a descriptive error (include which checks failed)
- Log warnings but don't fail for non-critical issues (e.g., slightly inconsistent winding on complex geometry)
  </action>
  <verify>docker compose run --rm backend python -c "
  from build123d import Box
  from app.modeler.export import export_stl_bytes
  from app.modeler.validation import validate_mesh, validate_dimensions

  r = validate_mesh(stl)
  print(f'Valid: {r.is_valid}, Watertight: {r.is_watertight}, Volume: {r.volume:.1f}')
  d = validate_dimensions(stl, {'width': 50, 'height': 30, 'depth': 10})
  print(f'Accurate: {d.is_accurate}')
