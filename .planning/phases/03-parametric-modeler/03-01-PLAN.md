Create the modeler package structure:

1. `modeler/__init__.py` — re-export `ModelEngine` and `generate_stl`
2. `modeler/engine.py` — `ModelEngine` class with:
   - `TEMPLATES: dict[str, Callable]` registry mapping category strings to template functions (empty for now, templates register in 03-03)
   - `generate(category: str, parameters: dict) -> bytes` — looks up template by category, calls it with parameters, gets Part, exports to STL bytes via export utility. Raises `ValueError` for unknown category. Validates output has non-zero bytes.
   - `register_template(category: str, fn: Callable)` — adds template to registry
3. `modeler/templates/__init__.py` — empty, will import templates in 03-03
4. `modeler/templates/base.py` — `TemplateProtocol` using `typing.Protocol`:
   - `def __call__(self, **params) -> "Part"` — takes keyword params, returns build123d Part
   - Document that all templates must accept keyword arguments matching their Pydantic model fields (minus category, units, notes)
5. `modeler/export.py` — `export_stl_bytes(part, tolerance=0.01, angular_tolerance=0.1) -> bytes`:
   - Uses `build123d.export_stl` with `io.BytesIO`
   - Returns bytes
   - Also add `validate_part(part) -> bool` that checks `part.volume > 0` and bounding box has positive extents

Use build123d algebra mode imports. All units are millimeters. Import typing.TYPE_CHECKING guard for Part type hint to avoid import errors outside Docker.
